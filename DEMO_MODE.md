# Demo 模式使用说明

## 什么是 Demo 模式？

Demo 模式允许你在不启动后端服务的情况下运行 MoFox UI，所有数据都使用模拟数据。这对于以下场景很有用：

- 快速预览 UI 界面
- 前端开发和调试
- 演示和展示
- 无需配置后端环境

## 如何启用 Demo 模式？

### 方法 1: 使用 demo 命令（推荐）

```bash
npm run demo
```

这会自动使用 `.env.demo` 配置文件启动前端。

### 方法 2: 修改 .env 文件

编辑 `.env` 文件，将 `VITE_DEMO_MODE` 设置为 `true`：

```env
VITE_DEMO_MODE=true
```

然后正常启动：

```bash
npm run dev
```

## Demo 模式功能

在 Demo 模式下：

✅ **可用功能：**
- 查看所有 UI 界面
- 浏览插件市场
- 查看模拟配置
- 查看模拟日志
- 查看模拟状态信息
- 所有交互和动画效果

⚠️ **限制：**
- 配置修改不会保存
- 日志数据是静态的
- 无法连接真实的 Bot 服务
- 数据在刷新后重置

## 构建 Demo 版本

如果需要构建一个 Demo 版本用于部署：

```bash
npm run build:demo
```

这会生成一个不依赖后端的静态网站。

## 切换回正常模式

### 方法 1: 使用正常命令

```bash
npm run start  # 启动完整版（前端+后端）
npm run dev    # 仅启动前端（需要后端已运行）
```

### 方法 2: 修改 .env 文件

将 `.env` 文件中的 `VITE_DEMO_MODE` 改回 `false`：

```env
VITE_DEMO_MODE=false
VITE_API_URL=http://127.0.0.1:8001
```

## 自定义 Demo 数据

如果需要修改 Demo 模式下的模拟数据，编辑以下文件：

```
src/utils/demoData.ts
```

你可以在这里修改：
- 配置数据
- 日志内容
- 状态信息
- 记忆数据

## 注意事项

1. Demo 模式下的所有操作都是模拟的，不会影响真实数据
2. 页面刷新后，所有"修改"都会重置
3. Demo 模式会在启动时显示提示信息
4. 插件市场功能仍然会从 GitHub 获取真实数据（需要网络连接）

## 故障排除

**问题：Demo 模式没有生效**
- 确认 `.env` 文件中 `VITE_DEMO_MODE=true`
- 重启开发服务器
- 清除浏览器缓存

**问题：看到后端连接错误**
- 检查是否正确设置了 Demo 模式
- 确认使用的是 `npm run demo` 命令

**问题：数据没有显示**
- 检查浏览器控制台是否有错误
- 确认 `src/utils/demoData.ts` 文件存在且格式正确
